# HIVE ATAK Plugin

ATAK plugin for managing HIVE (Hierarchical Intelligence for Versatile Entities) autonomous systems via the TAK ecosystem.

## Overview

This plugin provides integration between HIVE and ATAK (Android Tactical Assault Kit), enabling:

- **Cell Visualization**: Display HIVE cell formations on the ATAK map
- **Platform Tracking**: Show individual platform positions with status
- **Track Display**: View tracks detected by HIVE platforms
- **Command Interface**: Send commands to HIVE cells via TAK UI
- **CoT Integration**: Bidirectional Cursor-on-Target message handling

## Prerequisites

- **ATAK CIV SDK** (5.5+) - Download from [tak.gov](https://tak.gov)
- **Android Studio** Arctic Fox or later
- **Android NDK** (for native library compilation)
- **JDK 17+**

## Setup

### 1. Download ATAK SDK

1. Create an account at [tak.gov](https://tak.gov)
2. Download ATAK-CIV SDK
3. Extract to a known location

### 2. Configure Local Properties

```bash
cp local.properties.template local.properties
```

Edit `local.properties` with your paths:

```properties
sdk.dir=/path/to/Android/Sdk
atak.sdk.dir=/path/to/atak-civ-sdk

takDebugKeyFile=/path/to/debug.keystore
takDebugKeyFilePassword=android
takDebugKeyAlias=androiddebugkey
takDebugKeyPassword=android
```

### 3. Generate Signing Keys (if needed)

```bash
keytool -genkeypair -dname "CN=Android Debug,O=Android,C=US" \
  -validity 9999 -keystore debug.keystore \
  -alias androiddebugkey -keypass android -storepass android
```

### 4. Build HIVE FFI Libraries

From the repository root:

```bash
# Build for all Android architectures
cargo build -p hive-ffi --release --target aarch64-linux-android
cargo build -p hive-ffi --release --target armv7-linux-androideabi
cargo build -p hive-ffi --release --target x86_64-linux-android
cargo build -p hive-ffi --release --target i686-linux-android

# Copy libraries to plugin
cd atak-plugin
./gradlew copyHiveNativeLibs
./gradlew copyHiveBindings
```

### 5. Build Plugin

```bash
./gradlew assembleCivDebug
```

Output APK: `app/build/outputs/apk/civ/debug/app-civ-debug.apk`

## Installation

1. Copy the APK to your Android device
2. Install via `adb install` or file manager
3. Open ATAK - the HIVE plugin will load automatically

## Usage

### Accessing HIVE Manager

1. Tap the HIVE icon in the ATAK toolbar
2. The HIVE Manager panel opens on the side

### Tabs

- **Cells**: View active HIVE cells with status and capabilities
- **Tracks**: View tracks being maintained by HIVE
- **Settings**: Plugin configuration

### Map Visualization

- Cell markers show cell center positions
- Color indicates status (green=active, yellow=forming, red=offline)
- Tap a cell marker to view details

## Integration with HIVE FFI

The plugin uses `hive-ffi` Kotlin bindings generated by UniFFI:

```kotlin
// Load native library
System.loadLibrary("hive_ffi")

// Get HIVE version
val version = uniffi.hive_ffi.hiveVersion()

// Encode track to CoT XML
val cotXml = uniffi.hive_ffi.encodeTrackToCot(trackData)
```

## CoT Message Types

The plugin handles these CoT message types:

| HIVE Concept | CoT Type | Description |
|--------------|----------|-------------|
| Cell | `a-f-G-U-C` | Friendly ground unit |
| UAV Platform | `a-f-A-M-F-Q` | Friendly UAV |
| UGV Platform | `a-f-G-U-C` | Friendly ground combat |
| Track (hostile) | `a-h-G` | Hostile ground |
| Track (unknown) | `a-u-G` | Unknown ground |

## Development

### Building with Compose

The plugin uses Jetpack Compose via the `atak-compose` library:

```kotlin
implementation("com.dittofederal:atak-compose:0.0.4")
```

### Adding Features

1. Add UI in `HiveDropDownReceiver.kt` using Compose
2. Add map features in `overlay/` package
3. Add data models in `model/` package
4. Use `HiveMapComponent` for state management
