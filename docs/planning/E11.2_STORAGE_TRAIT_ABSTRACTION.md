# E11.2: Storage Trait Abstraction & RocksDB Implementation

**Epic**: ADR-011 Implementation (Automerge + Iroh Integration)
**Status**: 🚧 In Progress
**Timeline**: Week 3-4 (2025-11-12 to 2025-11-26)
**Priority**: Critical Path

---

## Goal

Create trait-based storage abstraction layer that enables runtime backend selection between Ditto, Automerge (in-memory), and RocksDB. This allows gradual migration while maintaining all existing functionality.

**Success Criteria**:
- All existing tests pass with Ditto backend (zero regressions)
- Runtime backend selection via environment variable
- Clean trait API suitable for future backends (RocksDB, Redb)

---

## Context

**From ADR-011 & Storage Due Diligence**:
- Decision: Replace Ditto with Automerge (CRDT) + open-source storage
- Storage choice: RocksDB for production (E11.2), Redb as future alternative
- Migration strategy: Trait abstraction → RocksDB impl → Gradual rollout

**Current State** (as of main branch):
- All storage goes through `DittoStore` (peat-protocol/src/storage/ditto_store.rs)
- Direct Ditto SDK calls throughout codebase
- No abstraction layer for swapping backends

**Why Traits First (Week 1)**:
- Enables testing both backends side-by-side
- Reduces risk (Ditto remains default)
- Allows incremental migration of callsites
- Clean separation of concerns

### Collection Trait Design

```
Business Logic needs:
  ├─ upsert(doc_id, data) → Store document
  ├─ get(doc_id) → Retrieve document
  ├─ delete(doc_id) → Remove document
  ├─ find(predicate) → Query documents
  ├─ scan() → Iterate all documents
  └─ geohash_prefix(prefix) → Proximity queries
```

## Technical Design

### 1. Storage Trait Definition

**File**: `peat-protocol/src/storage/traits.rs` (NEW)

```rust
use anyhow::Result;
use std::sync::Arc;

/// Main storage backend trait
///
/// Implementations: DittoStore, AutomergeInMemoryStore, RocksDbStore
pub trait StorageBackend: Send + Sync {
    /// Get or create a collection by name
    fn collection(&self, name: &str) -> Arc<dyn Collection>;

    /// List all collection names
    fn list_collections(&self) -> Vec<String>;

    /// Flush any pending writes to disk (no-op for some backends)
    fn flush(&self) -> Result<()>;

    /// Close the database cleanly
    fn close(self) -> Result<()>;
}

/// Collection trait for storing documents
///
/// Documents are serialized protobuf messages stored as bytes
pub trait Collection: Send + Sync {
    /// Insert or update a document
    fn upsert(&self, doc_id: &str, data: Vec<u8>) -> Result<()>;

    /// Get a document by ID
    fn get(&self, doc_id: &str) -> Result<Option<Vec<u8>>>;

    /// Delete a document by ID
    fn delete(&self, doc_id: &str) -> Result<()>;

    /// Scan all documents in collection
    fn scan(&self) -> Result<Vec<(String, Vec<u8>)>>;

    /// Find documents matching a predicate (for queries)
    fn find(&self, predicate: Box<dyn Fn(&[u8]) -> bool + Send>) -> Result<Vec<(String, Vec<u8>)>>;

    /// Query documents by geohash prefix (proximity queries)
    fn query_geohash_prefix(&self, prefix: &str) -> Result<Vec<(String, Vec<u8>)>>;

    /// Count documents in collection
    fn count(&self) -> Result<usize>;
}
```

### 2. Ditto Backend Implementation

**File**: `peat-protocol/src/storage/ditto_backend.rs` (NEW - wraps existing DittoStore)

```rust
use super::traits::{StorageBackend, Collection};
use super::ditto_store::DittoStore as InnerDittoStore;
use anyhow::Result;
use std::sync::Arc;

/// Ditto backend implementation (wraps existing DittoStore)
pub struct DittoBackend {
    inner: Arc<InnerDittoStore>,
}

impl DittoBackend {
    pub fn new(config: &DittoConfig) -> Result<Self> {
        let inner = InnerDittoStore::new(config)?;
        Ok(Self {
            inner: Arc::new(inner),
        })
    }
}

impl StorageBackend for DittoBackend {
    fn collection(&self, name: &str) -> Arc<dyn Collection> {
        Arc::new(DittoCollection {
            store: self.inner.clone(),
            collection_name: name.to_string(),
        })
    }

    fn list_collections(&self) -> Vec<String> {
        // Ditto doesn't expose this directly, return known collections
        vec![
            "cells".to_string(),
            "nodes".to_string(),
            "capabilities".to_string(),
            "squad_summaries".to_string(),
            "platoon_summaries".to_string(),
        ]
    }

    fn flush(&self) -> Result<()> {
        // Ditto handles flushing automatically
        Ok(())
    }

    fn close(self) -> Result<()> {
        // Ditto SDK handles cleanup on drop
        Ok(())
    }
}

/// Ditto collection implementation
struct DittoCollection {
    store: Arc<InnerDittoStore>,
    collection_name: String,
}

impl Collection for DittoCollection {
    fn upsert(&self, doc_id: &str, data: Vec<u8>) -> Result<()> {
        // Delegate to existing DittoStore methods
        // Deserialize protobuf, call appropriate store method
        match self.collection_name.as_str() {
            "cells" => {
                let cell_state = CellState::decode(&data[..])?;
                self.store.upsert_cell(&cell_state)?;
            }
            "nodes" => {
                let node = Node::decode(&data[..])?;
                self.store.upsert_node(&node)?;
            }
            "capabilities" => {
                let capability = Capability::decode(&data[..])?;
                self.store.upsert_capability(&capability)?;
            }
            "squad_summaries" => {
                let summary = SquadSummary::decode(&data[..])?;
                self.store.store_squad_summary(&summary)?;
            }
            "platoon_summaries" => {
                let summary = PlatoonSummary::decode(&data[..])?;
                self.store.store_platoon_summary(&summary)?;
            }
            _ => return Err(anyhow!("Unknown collection: {}", self.collection_name)),
        }
        Ok(())
    }

    fn get(&self, doc_id: &str) -> Result<Option<Vec<u8>>> {
        // Delegate to existing DittoStore methods
        match self.collection_name.as_str() {
            "cells" => {
                if let Some(cell) = self.store.get_cell(doc_id)? {
                    let bytes = cell.encode_to_vec();
                    Ok(Some(bytes))
                } else {
                    Ok(None)
                }
            }
            "nodes" => {
                if let Some(node) = self.store.get_node(doc_id)? {
                    let bytes = node.encode_to_vec();
                    Ok(Some(bytes))
                } else {
                    Ok(None)
                }
            }
            // ... similar for other collections
            _ => Err(anyhow!("Unknown collection: {}", self.collection_name)),
        }
    }

    fn delete(&self, doc_id: &str) -> Result<()> {
        // Delegate to existing DittoStore methods
        match self.collection_name.as_str() {
            "cells" => self.store.delete_cell(doc_id),
            "nodes" => self.store.delete_node(doc_id),
            // ... similar for other collections
            _ => Err(anyhow!("Unknown collection: {}", self.collection_name)),
        }
    }

    fn scan(&self) -> Result<Vec<(String, Vec<u8>)>> {
        // Delegate to existing DittoStore query methods
        match self.collection_name.as_str() {
            "cells" => {
                let cells = self.store.query_all_cells()?;
                Ok(cells.into_iter()
                    .map(|cell| (cell.id.clone(), cell.encode_to_vec()))
                    .collect())
            }
            // ... similar for other collections
            _ => Err(anyhow!("Unknown collection: {}", self.collection_name)),
        }
    }

    fn find(&self, predicate: Box<dyn Fn(&[u8]) -> bool + Send>) -> Result<Vec<(String, Vec<u8>)>> {
        // Scan and filter
        let all = self.scan()?;
        Ok(all.into_iter().filter(|(_, data)| predicate(data)).collect())
    }

    fn query_geohash_prefix(&self, prefix: &str) -> Result<Vec<(String, Vec<u8>)>> {
        // Delegate to DittoStore's geohash query
        match self.collection_name.as_str() {
            "nodes" => {
                let nodes = self.store.query_nodes_by_geohash_prefix(prefix)?;
                Ok(nodes.into_iter()
                    .map(|node| (node.id.clone(), node.encode_to_vec()))
                    .collect())
            }
            _ => Err(anyhow!("Geohash queries only supported for nodes collection")),
        }
    }

    fn count(&self) -> Result<usize> {
        Ok(self.scan()?.len())
    }
}
```

### 3. Backend Factory

**File**: `peat-protocol/src/storage/backend.rs` (NEW)

```rust
use super::traits::StorageBackend;
use super::ditto_backend::DittoBackend;
use anyhow::{Result, anyhow};
use std::sync::Arc;

/// Storage backend configuration
#[derive(Clone)]
pub struct StorageConfig {
    /// Backend type: "ditto", "automerge-memory", "rocksdb"
    pub backend: String,
    /// Data directory path (for persistent backends)
    pub data_path: Option<String>,
    /// Ditto-specific configuration
    pub ditto_config: Option<DittoConfig>,
}

impl StorageConfig {
    /// Load configuration from environment variables
    pub fn from_env() -> Result<Self> {
        let backend = std::env::var("CAP_STORAGE_BACKEND")
            .unwrap_or_else(|_| "ditto".to_string());

        let data_path = std::env::var("CAP_DATA_PATH").ok();

        // Load Ditto config if backend is ditto
        let ditto_config = if backend == "ditto" {
            Some(DittoConfig::from_env()?)
        } else {
            None
        };

        Ok(Self {
            backend,
            data_path,
            ditto_config,
        })
    }
}

/// Create storage backend from configuration
pub fn create_storage_backend(config: &StorageConfig) -> Result<Arc<dyn StorageBackend>> {
    match config.backend.as_str() {
        "ditto" => {
            let ditto_config = config.ditto_config.as_ref()
                .ok_or_else(|| anyhow!("Ditto config required for ditto backend"))?;
            Ok(Arc::new(DittoBackend::new(ditto_config)?))
        }
        "automerge-memory" => {
            // Will implement in E11.2 Week 2
            todo!("Automerge in-memory backend not yet implemented")
        }
        "rocksdb" => {
            // Will implement in E11.2 Week 2
            todo!("RocksDB backend not yet implemented")
        }
        other => Err(anyhow!("Unknown storage backend: {}", other)),
    }
}
```

## Risks & Mitigations

### Risk 1: Trait Design Not Flexible Enough

**Likelihood**: Medium
**Impact**: High (would require redesign)

**Mitigation**:
- Review trait design with team before implementation
- Reference existing storage abstractions (sled, redb APIs)
- Keep traits minimal (add methods later if needed)
- Use `Box<dyn Fn>` for predicates (flexibility)

### Risk 2: Performance Overhead from Trait Objects

**Likelihood**: Low
**Impact**: Medium

**Mitigation**:
- Use `Arc<dyn Trait>` to minimize allocation
- Benchmark before/after trait abstraction
- If overhead exists, acceptable for migration period
- Can optimize later with generics if needed

### Risk 3: Ditto API Doesn't Map Cleanly to Trait

**Likelihood**: Medium
**Impact**: Medium

**Mitigation**:
- DittoBackend can be "thick wrapper" with business logic
- Trait methods can return `Vec<u8>` (serialized protobuf)
- Collection-specific logic in wrapper (not trait)
